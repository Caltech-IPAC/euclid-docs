class SIR_RFXModel.FPA_Geometry.GlobalSettings[source]

Bases: object

Class containing global settings.

The properties are:

  • pairCoordXY (boolean, default False): whether the coordinate pairs of pixel should be interpreted as first and second index in the 2D array (as read with astropy.io.fits);

  • enableDetRotation (boolean, default False): set to True to enable rotation of detectors in the bottom part of the focal plane;

  • binning (int, default 3): pixel binning factor to display images. Must be a positivie integer number smaller than 9. Increase this number to improve performances;

  • markpixels (boolean, default False): set to True to mark reference pixels in the image. Note: this will erase all pixel information;

  • markplot (boolean, default False): set to True to mark reference pixels in the plot.

pix2fp(det, x, y)[source]

@brief Convert the location of pixel(s) in a detector to the corresponding position(s) in the focal plane.

@param det: a scalar detector name (among the allowed ones, see detnames()); @param x: list or numpy array of pixel coordinates along horizontal direction; @param y: list or numpy array of pixel coordinates along vertical direction;

@returns 2-element tuple of N-element lists, each containing the locations on the focal plane (in mm) along the horizontal and vertical directions respectively of the N pixel coordinates provided as input.

Note: the pixel coordinates are supposed to be in the range 0:det_npix_horiz-1 or 0:det_npix_vert-1. However, it is possible to provide numbers outside this limit, to obtain positions in the focal plane corresponding to the gaps between detectors.

Example: from NIR_misc_utils.FocalPlaneModel import * fpmodel = FPModel(‘NISPDetectorSlots.csv’, 2048, 2048) fp_x, fp_y = fpmodel.pix2fp(‘33’, [3, 2043], [3, 2043]) # ([1.601895000000006, 38.160735], [2.704857000000004, 397.683697])

SIR_RFXModel.SIR_RelFluxModelCalibration.mainMethod(args)[source]

Main entry point for the SIR relative flux calibration.

This method wraps the end-to-end workflow:

  1. Parse configuration and command-line options.

  2. Build the self-calibration data sets for the selected mode.

  3. Run the SelfCalibWave solver to derive the relative flux cubes.

  4. Write the Euclid-compliant calibration product to disk.
