[docs]class NestedDict(object):
    """
    A Convenience class for managing nested dictionaries.

    The NestedDict provides a standardized way to handle nested dictionaries, which are tree-like data structures of
    the form:

        {field_0: {field_1: {field_2: {... {field_N: item}}}

    The fields of the nested dictionary are named (i.e., they are strings), their order in the nesting is specified upon
    construction. The type of object that will be stored at the leaves of the nested dict is expected to be uniform
    throughout the nested dict and is also specified upon construction (instantiation).

    Such nested dictionaries are useful for organizing hierarchical data for certain data access patterns, however they
    become cumbersome to deal with when the depth of the nesting exceeds 2. For example, suppose we had a nested
    dictionary storing information about automobiles. It is of the format:

        {manufacturer: {model: {year: [VIN]}}}

    where
        * manufacturer: the name of the company that manufactured the automobile.
        * model: the name of the model of the automobile
        * year: the model year of the automobile
        * [VIN]: a list of Vehicle Identification Numbers

        For example, to obtain the list of VINs for 1998 Toyota Avalons, we would use the syntax:

            vins = vin_numbers['Toyota']['Avalon'][1998]

        This might not seem too bad, but what if you forget the order in which the nesting was performed? It could have
        just as easily have been organized as[1998]['Toyota']['Avalon']. If you forget how the data is structured, then
        you have to inspect the contents or refer to documentation.

        Another problem occurs when you wish to add new data to the data structure or fetch all of the values in a
        sub-portion of the data structure.

        With the NestedDict class, we can define the structure of the vin_numbers nested dictionary like this:

            vin_numbers = NestedDict(list, 'manufacturer', 'model', 'year')

        To access the VIN numbers of the 1998 Toyota Avalons, we could do any of the following:

            vin_numbers['Toyota', 'Avalon', 1998]

            vin_numbers.get_item('Toyota', 'Avalon', 1998)

            vin_number.get_item(year=1998, manufacturer='Toyota', model='Avalon')

        Notice that the third option does not require knowledge of the order of the nesting.

        In order to find out how the data is nested, we can use the `fields` property, as in:

           vin_number.fields

        If we need to add a list of VINs for a new type of automobile, we simply use:

           vin_number['Tesla', 'Model 3', 2017] = vins

        or

           vin_number.add_item(vins, 'Tesla', 'Model 3', 2017)

        or

           vin_number.add_item(vins, manufacturer='Tesla', year=2017, model='Model 3')

        All parent dictionaries are automatically created. If we try to add an object of the wrong type or we try to
        increase the depth of the nested dictionary, we encounter an error (exception).

        If we wanted to obtain the VINs for all model years of Hyundai Sonatas, we could do this:

            vin_number.values('Hyundai', 'Sonata')

        or
            vin_number.values(model='Sonata', manufacturer='Hyundai')

        If we wanted to obtain the nested dictionary containing all Honda Accord vehicles:

           vin_number.get_subdict('Honda', 'Accord')

        This returns a regular dict object. To return a NestedDict containing information for all Hondas, we would do
        this:

            honda = vin_number.get_nested_dict('Honda')

        or
            honda = vin_number.get_nested_dict(manufacturer='Honda')

        Note that we cannot return a NestedDict containing all Honda Accords, because the resulting dictionary would
        not be nested (it would be of the format {year: [VIN]}).

        Since `honda` is a sub-dictionary of vin_number, it has fewer fields. In particular, honda.fields returns
        ('model', 'year').

        We can also return all items, along with their paths using the `items` property. For example:

            for vin in vin_number.items:
                print(vin)

        would print out something like this:

            ('Acura', 'ILX', 2018, [list of VINS])
            ('Acura', 'ILX', 2017, [list of VINS])
            ('Acura', 'ILX', 2016, [list of VINS])
            .
            .
            .
            ('Volvo', 'PV51-7', 1936, [list of VINS])

    """
    # TODO: consider adding the ability to ingest a raw nested dict.
    def __init__(self, dtype: type, *fields, plural_key=None, plural_value=None):
        """
        Constructs an instance of the nested dictionary class.

        Parameters
        ----------

        dtype: type
            The type of object that is stored in the nested dict (there should be only one type). For example, if the
            structure of the nested dict is `{country_code : {postal_code : [list_of_names]}}`, then the data type is
            `list`, since calling `dict[country_code][postal_code]` returns a list of names.

        fields: tuple
            The function accepts a variable number of arguments which are automatically packed into the `fields` tuple.
            The fields correspond to the names of the fields in the nested dict. For example, if the constructor is
            provided with the parameters `('country_code', 'postal_code')`, then the structure of the nested dictionary
            would be `{country_code : {postal_code : value}}`. Fields must be unique and they must be valid Python
            identifiers (valid variable names).

        plural_key: str (optional)
            The plural name of the first field in the list of fields. If provided, this allows for more readable code by
            allowing the use of a more meaningful name when iterating over the keys. Suppose the first field in the
            `fields` is 'country'. The normal way to iterate over the outermost loop would be:
            `for country in nd.keys()`, but if `plural_key='countries'` is provided upon construction, then
            this can be written as `for country in nd.countries()`, which is more readable.

        plural_value: str (optional)
            The plural name of the final field in the list of fields (i.e., the quantity whose type is specified by
            `dtype`). This creates an alias to the `.values()` method, similar to the way that plural_key causes the
            creation of an alias for `.keys()`.


        Raises
        ------

        ValueError:
            Raised if an insufficient number of parameters are provided or if the fields are not unique or if the names
            of the fields are illegal.

        TypeError:
            Raised if the first argument does not specify a data type or if one of the field .
        """
        if not isinstance(dtype, type):
            raise TypeError('The dtype must specify a data type.')

        if issubclass(dtype, dict):
            raise TypeError('The data type of the leaves cannot be dict')

        if len(fields) < 2:
            raise ValueError('The NestedDict requires at least two field names.')

        for field in fields:
            if not isinstance(field, str):
                raise TypeError('All field names must be strings.')

            if not field.isidentifier():
                raise ValueError(f'The fields must all be valid Python identifiers. "{field}" is not valid.')

        if len(set(fields)) != len(fields):
            raise ValueError('The fields must be unique; field names cannot be repeated.')

        self._fields = fields
        self._indices = {name: i for i, name in enumerate(fields)}
        self._root = {}
        self._dtype = dtype

        if plural_key is not None and isinstance(plural_key, str):
            self._make_iterator_alias('keys', plural_key)

        if plural_value is not None and isinstance(plural_value, str):
            self._make_iterator_alias('values', plural_value)

        self._plural_val = plural_value

    @property
    def fields(self):
        """The names of the fields of the nested dictionary"""
        return self._fields

    @property
    def root(self):
        """The highest-level (outer-most) dict in the nested dictionary."""
        return self._root

    @property
    def dtype(self):
        """The data type of the items stored in the dictionary."""
        return self._dtype

    @property
    def items(self):

        Returns
        -------
        iterable
            A generator which returns tuples of the form (key0, key1, key2, ... keyN, value)
        """
        return self._get_flat()

        """
        Finds the index of the specified field.

        Parameters
        ----------

        name: str
            The name of a field in the nested dict.

        Returns
        -------

        index: int
            The index of the field within the nested dictionary. For example, if the structure of the nested dict is
            `{country code : {postal code: [list of names]}}`, then the dict is indexed as
            `dict[country code][postal code]`. Thus `index_of('postal code')` returns 1 and `index_of('country code')`

        """
        return self._indices.get(name)


    def __eq__(self, other):
        """
        Tests for equality with another NestedDict

        Parameters
        ----------

        other: NestedDict
            The NestedDict that will be checked for equality with the current NestedDict.

        Returns
        -------

        Bool
            True if `other` contains the same nested dictionary and has the same field names as this nested dictionary.
            False otherwise.
        """
        if not isinstance(other, self.__class__):
            return False

        return self.dtype == other.dtype and self.fields == other.fields and self.root == other.root

    def __ne__(self, other):
        """
        Parameters
        ----------

        other: NestedDict
            The NestedDict that will be checked for equality with the current NestedDict.

        Returns
        -------

        Bool
            False if `other` contains the same nested dictionary and has the same field names as this nested dictionary.
            True otherwise.
        """
        return not self.__eq__(other)

            The object stored in the nested dict at the specified location. If the path does not exist, returns None.

        self._check_path(path)
        self.delete(*path)

    def __str__(self):
        return str(self.root)

    def __repr__(self):
        fields = "', '".join(self.fields)
        string = f"NestedDict({self.dtype}, '{fields}'):\n    {self}"
        return string


[docs]    def get_item(self, *path, **kwpath):
        """
        Returns the item located at the specified location within the full nested dict.

        Parameters
        ----------

        path: tuple
            The location in the nested dict of a sub-dict.

        kwpath: dict:
            The location in the requested item as a dictionary.

        Returns
        -------

        item: self.dtype
            The item located at the specified location

        Raises
        ------

        KeyError
           Raised if the path and kwpath are both specified; only one is allowed.

        ValueError
           Raised if the kwpath does not represent a path.
        """

        if len(kwpath) > 0 and len(path) > 0:
            raise KeyError('The path must either be specified as an ordered tuple or using field_name=key.')

        if len(kwpath) > 0:
            path = self._to_path(**kwpath)

        self._check_path(path)

        current_dict = self._root

        for key in path[:-1]:
            item = current_dict.get(key)
            if isinstance(item, dict):
                current_dict = item

        item = current_dict.get(path[-1])

        if isinstance(item, self._dtype):
            return item
        else:
            return None



[docs]    def add_item(self, item, *path, **kwpath):
        """
        Adds an item to the nested dict at the location specified by `path`.

        Parameters
        ----------

        item: self.dtype
            The item that will be added to the nested dictionary.

        path: tuple
            The location in the nested dict in which to place the item. If the location does not exist, it will be
            created. If an item already exists in the specified location, it will be replaced by the new `item`.

        kwpath: dict:
            The location in the nested dict. The structure of the kwpath should be
            `{field_name_0: key_0, field_name_1: key_1, ... field_name_N: key_N}`. This allows  the user to call the
            function like `.keys(field_name_N=key_N, field_name_1=key_1, ...)` so that the user does not need to
            remember the order in which the dictionaries corresponding to the fields are nested.

        Side Effects
        ------------

        Modifies the internal nested dictionary.

        Raises
        ------

        KeyError
            Raised if the dimensionality of the path is incorrect or if the path and kwpath are both specified; only
            one is allowed.

        TypeError
            Raised if the data type of the `item` does not match the expected type specified by `self.dtype`.

        ValueError
           Raised if the kwpath does not represent a path.
        """
        # handle the special case in which an int is assigned to a float:

        if self._dtype == float and isinstance(item, int):
            item = float(item)
        else:
            self._check_type(item)

        if len(kwpath) > 0 and len(path) > 0:
            raise KeyError('The path must either be specified as an ordered tuple or using field_name=key.')

        if len(kwpath) > 0:
            path = self._to_path(**kwpath)

        self._check_path(path)

        current_dict = self._root

        for key in path[:-1]:
            if key not in current_dict:
                current_dict[key] = {}
            current_dict = current_dict[key]

        current_dict[path[-1]] = item



[docs]    def delete(self, *path):
        """
        Deletes the sub-dictionary or item located at the specified path.

        Parameters
        ----------

        path: tuple
            The location of the item or sub-dict that will be deleted.
        """
        if len(path) == 0:
            del self._root
            self._root = {}
            return

        current_dict = self._root

        for key in path[:-1]:
            item = current_dict.get(key)
            if isinstance(item, dict):
                current_dict = item
            elif item is None:
                return

        if path[-1] in current_dict:
            del current_dict[path[-1]]

        # clean up empty parent dictionaries.

        if len(self.get_subdict(*path[:-1])) == 0:
            self.delete(*path[:-1])

        if not isinstance(subdict, dict):

[docs]    def get_subdict(self, *path, **kwpath):
        """
        Returns the sub-dictionary located at the specified location within the full nested dict.

        Parameters
        ----------

        path: tuple
            The location in the nested dict of a sub-dict.

        kwpath: dict:
            The location in the nested dict of a sub-dict. The structure of the kwpath should be
            `{field_name_0: key_0, field_name_1: key_1, ... field_name_N: key_N}`. This allows the user to call the
            function like `.keys(field_name_N=key_N, field_name_1=key_1, ...)` so that the user does not need to
            remember the order in which the dictionaries corresponding to the fields are nested.

        Returns
        -------

        subdict: dict or dtype
            The dictionary located at the specified location. If no path is provided, then this is the same as the root
            dictionary. Returns None if the format of the path is correct but it does not point to an existing location.
            If the path points to a leaf, then the item at the leaf is returned.

        Raises
        ------

        KeyError
           Raised if the path and kwpath are both specified; only one is allowed or if the path specifies an item rather
           than a sub-dictionary.

        ValueError
           Raised if the kwpath does not represent a path.
        """

        if len(kwpath) > 0 and len(path) > 0:
            raise KeyError('The path must either be specified as an ordered tuple or using field_name=key.')

        if len(kwpath) > 0:
            path = self._to_path(**kwpath)

        if len(path) == 0:
            return self._root

        if len(path) == len(self.fields):
            return self.get_item(*path)

        current_dict = self._root

        for key in path:
            if current_dict is not None:
                item = current_dict.get(key)
                if item is None:
                    current_dict = None
                if isinstance(item, dict):
                    current_dict = item

        if item is current_dict:
            return current_dict



[docs]    def get_nested_dict(self, *path, plural_key=None, **kwpath):
        sub = self.get_subdict(*path, **kwpath)
        depth = max(len(path), len(kwpath))
        return self._nested_subdict(sub, depth, plural_key)


    def _to_path(self, **kwpath):
        """
        Converts the location of an object, specified as field names and keys, to a properly ordered path (tuple of
        keys).

        Parameters
        ----------

        kwpath: dict
            The path of a location within the nested dictionary. The structure of the kwpath should be
            `{field_name_0: key_0, field_name_1: key_1, ... field_name_N: key_N}`. This allows
            the user to call the function like `.keys(field_name_N=key_N, field_name_1=key_1, ...)` so that the
            the user does not need to remember the order in which the dictionaries corresponding to the fields are
            nested.

        Returns
        -------

        path: tuple
            A tuple of the keys, ordered according to the structure of the nested dict (the order given by the `fields`
            property).

        Raises
        ------

        ValueError
            Raised if the entries in the kwpath dict do not describe a complete path within the nested dictionary.
        """
        if len(kwpath) > len(self.fields):
            raise ValueError('Could not convert to a path; too many keywords were provided.')

        path_list = [None for _ in range(len(self.fields))]

        for field_name, key in kwpath.items():
            idx = self.index_of(field_name)
            if idx is not None:
                path_list[idx] = key
            else:
                raise ValueError(f"{field_name} is an invalid field name.")

        # At this point, we have a list of keys in the correct order, but there may be some None entries in the list.
        # The None entries need to be removed if they are at the end of the list. If they are not at the end of the list
        # then the path is invalid.

        # clear trailing Nones:

        encountered_value = False

        for i in reversed(range(len(path_list))):
            key = path_list[i]

            if key is None and not encountered_value:

                del path_list[i]
            else:
                encountered_value = True

        if any(map(lambda x: x is None, path_list)):
            raise ValueError(f"The provided keys do not describe a complete path. The equivalent path is {path_list}.")
        else:
            return tuple(path_list)

    def _check_type(self, item):
        if not isinstance(item, self.dtype):
            raise TypeError(f'Expected a {self.dtype}, but encountered a {type(item)}')

    def _check_path(self, path: tuple):
        if (not isinstance(path, tuple)) or len(path) != len(self._fields):
            raise KeyError('Expected the dimensionality of the key to match the number of dimensions of the nested '
                           f'dictionary ({len(self._fields)} in this case).')

        for key in path:
            if isinstance(key, slice):
                raise KeyError("slices are currently not supported.")

    def _nested_subdict(self, subdict, depth, plural_key=None):
        """
        Returns the sub-dictionary located at the specified location within the full nested dict.

        Parameters
        ----------

        subdict: dict
            The sub-dict (regular python dict) that will be the root of the new NMestedDict.

        depth: int
            The depth of the subdict within the nested dict

        plural_key: str
            The plural name of the new root field.

        Returns
        -------

        keys: dict
            The dictionary located at the specified location.

        Raises
        ------

        KeyError
           Raised if the path and kwpath are both specified; only one is allowed.

        ValueError
           Raised if the kwpath does not represent a path.
        """
        nested_sub = NestedDict(self.dtype, *self.fields[depth:], plural_key=plural_key, plural_value=self._plural_val)
        nested_sub._root = subdict

        return nested_sub

    def _get_flat(self, path: tuple=()):
        subdict = self.get_subdict(*path)

        if isinstance(subdict, dict):
            for key, value in subdict.items():
                if isinstance(value, dict):
                    sub_path = join(*path, key)
                    for v in self._get_flat(sub_path):
                        yield v
                else:
                    # value is a leaf (value)
                    yield join(*path, key, value)
        else:
            # subdict is a leaf (value)
            yield join(*path, subdict)

    def _make_iterator_alias(self, iterator_type: str, plural_name: str):
        """
        Adds member functions to the object, for accessing the keys and values by name.

        Adds member functions that can be used essentially as aliases for .keys() or .values() to allow
        the user's code to be more readable.

        Parameters
        ----------

        iterator_type: str
            'keys' or 'values'

        plural_name: str
            The name that will be used as the alias for the objects stored in the iterator. The name should be
            plural, so if the field name is 'house', the plural_name should be 'houses'.

        Raises
        ------

        ValueError: if the specified name is already taken or if the iterator type is invalid.

        Side Effects
        ------------

        Adds a new method, `plural_name()`, to the instance.

        """
        if iterator_type not in ('keys', 'values'):
            raise ValueError(f'iterator_type must be either "keys" or "values"---not {iterator_type}')

        if plural_name in self.__dir__():
            raise ValueError(f'The name {plural_name} conflicts with the NestedDict API.')

        if not plural_name.isidentifier():
            raise ValueError(f'The name {plural_name} is not a valid identifier.')

        def get_keys(self):
            return (key for key in self.keys())

        def get_values(self):
            return self.values()

        iterator_func = get_keys if iterator_type == 'keys' else get_values

        self.__dict__[plural_name] = types.MethodType(iterator_func, self)

    @staticmethod
    def _values_of_dict(dictionary: dict):
        """
        Recursively produces a list of all of the items stored within the dictionary, `dictionary`.

        Parameters
        ----------

        dictionary: dict
            The nested dictionary whose values are of interest.

        Returns
        -------

        total: generator
            A generator that returns all of the objects stored in the specified dictionary.
        """
        for key, value in dictionary.items():
            if isinstance(value, dict):
                for v in NestedDict._values_of_dict(value):
                    yield v
            else:
                yield value
